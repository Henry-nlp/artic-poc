Related Work – HMM

  1. Machine Learning technique used to solve sequence labeling problems
  2. Metadata extraction as an instance of the sequence labeling problem
  3. HMM: Label each word and/or line with a class, such as: title, author
  4. Build the model using a training set
  5. Test the model using a test set
  6. Seymore et al. (1999), Yin et al. (2004)

Related Work – CRF

  1. HMM requires the enumeration of all possible observations
  2. HMM assumes that the observations are independent
  3. CRF overcomes HMM limitations for sequence labeling problems

CRF – Luong et al. (2010)

  2. Build the features to allow the CRF to identify the classes (metadata)
  3. Used OmniPage OCR tool to extract the PDF features

ARTIC: Contributions

  1. Two-layer CRF model: line-level (first layer), word-level (second layer)
  2. Relationship between authors, emails, and affiliations (post-processing)
  3. Output in a well-defined format (JSON)

ARTIC: Overview

ARTIC: Overview

ARTIC: First-level

  1. Header: information about the conference
  2. Title: title of the paper =D
  3. Author Information: author name, emails, and affiliations
  4. Body: non-relevant data for metadata extraction
  5. Footnote: non-relevant data for metadata extraction
  6. Some Features:

    7. Alignment, Bold, Underline, Italic, and Font Size

ARTIC: Second-level

Header CRF: explores the words of the lines with Header class (first-level).

  1. Conference Name: conference/journal name that the paper has been published
  2. Conference Year: year when the conference happened
  3. Conference Date: date when the conference happened
  4. Conference Location: location where the paper has been published
  5. Publisher: ACM, IEEE, Springer, Elsevier
  6. Other: non-relevant information for metadata extraction
  7. Some Features: Word Content, Character Length, Month, Year, and Country

ARTIC: Second-level

Author Information CRF: explores the words of the lines with Author Information class (first-level).

  1. Author Name: author name (words without relationship)
  2. Affiliation: author's affiliation (words without relationship)
  3. Email: author's email (words without relationship)
  4. Other: non-relevant information for metadata extraction
  5. Some Features: Word Content, Possible Affiliation, and Possible Email

ARTIC: Second-level

Footnote CRF: explores the words of the lines with Footnote class (first-level).

  1. Conference Name, Conference Year, Conference Data, Conference Location, Publisher, Email, Other
  2. ISBN: ISBN number (e.g.: 978-1-60558-01/08/04)
  3. Some features: Word Content, Possible Email, and ISBN

ARTIC: Post-processing

Conclusions

Conclusion

  1. OCR as the most effective way to extract rich text information
  2. OmniPage runs only on Windows :/
  3. If we use OCR, we support what they support, not only PDF =D
  4. 100 papers still small compared to the size of papers available online
  5. Possible Future Work: evaluate tools to minimize manual annotation

Conclusion

  1. No machine learning in the Post-Processing algorithms
  2. DO et al. (2013) presents a method to match authors and affiliations using Support Vector Machine (SVM)
  3. Possible Future Work: evaluate the use of ML techniques also at the post-processing level

Conclusion

  1. Artic presented a relative gain of 6.92% compared to SectLabel.
  2. The post-processing algorithms represented a relative loss of 5.34% compared to Artic classification step.
  3. Artic has been released as an open-source project at https://github.com/alansouzati/artic-poc.
  4. A paper has been submitted to DocEng 2014 conference which will be held in Fort Collins, USA and has a Qualis of B1
